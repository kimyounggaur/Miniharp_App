# 🪕 미니하프 연주 웹앱 — 통합 바이브코딩 프롬프트 (FINAL · 15/19/23현)

> 브라우저에서 키보드와 마우스/터치로 15현, 19현, 23현 미니하프를 연주하는 인터랙티브 웹앱을 개발합니다.
> 외부 라이브러리 없이 순수 HTML/CSS/Vanilla JS와 Web Audio API만 사용하여 **단일 `index.html` + `assets/` 폴더** 구조로 완성해 주세요.

## 1. 이미지 자산 및 화면 규격

**이미지 자산 파일명 매핑 규칙**
제공된 원본 이미지를 `assets/` 폴더에 아래 이름으로 저장하여 사용합니다.
- `assets/harp-15.jpg` (15현 하프 원본)
- `assets/harp-19.jpg` (19현 하프 원본)
- `assets/harp-23.jpg` (23현 하프 원본)

**화면 및 UI/UX 규격**
- **9:16 풀스크린 비율**: 앱의 루트(`#app`)는 모바일 화면을 가득 채우고(`100dvh`), 데스크톱에서는 9:16 비율의 중앙 패널 형태로 표시됩니다. 
- **미니멀 디자인**: UI는 Notion 모바일 앱이나 Figma 프로토타입처럼 군더더기 없이 깔끔하고 모던하게 디자인합니다. 부드러운 곡선과 옅은 그림자, 직관적인 버튼을 사용하세요.
- **이미지 무왜곡**: 하프 이미지는 절대 종횡비를 변형하지 않고 `object-fit: contain`으로 표시합니다.

## 2. 화면 구성

- **화면 1 (선택 화면)**: 15현, 19현, 23현 미니하프 중 하나를 선택할 수 있는 3개의 세로형 카드 UI 제공.
- **화면 2 (연주 화면)**:
  - 상단: `← 뒤로가기`, 선택된 하프 이름, 볼륨 조절 슬라이더.
  - 중앙 메인: 9:16 영역 내에 하프 이미지를 최대한 크게 띄우고, 그 위에 투명한 **현(String) 형태의 히트박스 레이어**를 배치.
  - 하단: 키보드 단축키 안내 패널 ("숫자=가운데음 · ⇧+숫자=높은음 · ⌃+숫자=낮은음").

## 3. 키보드 입력 규칙 및 음역 (핵심 요구사항)

사용자의 키보드 입력은 **수식키**를 통해 세 가지 음역으로 나뉩니다.
- `Ctrl` + 숫자키 (1~7): **B3 이하 낮은음** (C3, D3, E3, F3, G3, A3, B3)
- `숫자키` 단독 (1~7): **C4~B4 가운데음** (C4, D4, E4, F4, G4, A4, B4)
- `Shift` + 숫자키 (1~9): **C5 이상 높은음** (C5, D5, E5, F5, G5, A5, B5, C6, D6)
  * *참고: 숫자 8은 C6, 9는 D6로 매핑.*

**각 모델별 지원 음역 매핑**
해당 악기에 실제로 존재하는 음만 소리가 나야 하며, 없는 키보드 조합은 무음 처리합니다. (`event.code` 기준 판별, `Alt`/`Meta` 눌림 무시, `event.repeat` 무시)

1. **15현 하프** (`G3 ~ G5`)
   - 15음: `G3, A3, B3` (Ctrl+5~7), `C4~B4` (1~7), `C5~G5` (Shift+1~5)
2. **19현 하프** (`F3 ~ C6`)
   - 19음: `F3, G3, A3, B3` (Ctrl+4~7), `C4~B4` (1~7), `C5~B5` (Shift+1~7), `C6` (Shift+8)
3. **23현 하프** (`C3 ~ D6`)
   - 23음: `C3~B3` (Ctrl+1~7), `C4~B4` (1~7), `C5~B5` (Shift+1~7), `C6, D6` (Shift+8, 9)

## 4. 투명 현(String) 오버레이 렌더링 및 좌표

하프의 현은 원형이 아니라 **길고 얇은 사선** 형태입니다. 
- `.harp-stage` 안에 이미지 비율에 맞춘 래퍼를 두고, 각 음표에 해당하는 `.string-hit` `<button>` 엘리먼트를 생성합니다.
- `width`는 터치/클릭이 쉽도록 약 4~6% 정도로 넓게 잡고, 실제 시각적인 현의 떨림은 내부의 `::after` 가상 요소(가느다란 선)로 표현합니다.
- 각 현은 `x`, `y`, `height`, `rotate` (기울기) 백분율 좌표 객체를 갖습니다. (예: `C4: { x: 45.0, y: 50.0, h: 60.0, rot: '-10deg' }`). 시작값은 임의로 배분하되, 왼쪽이 긴 저음, 오른쪽이 짧은 고음으로 시계방향/사선 배열을 이룹니다.
- 마우스 클릭, 터치, 키보드 입력 시 해당 엘리먼트에 `.is-playing` 클래스가 추가되며, 현이 좌우로 미세하게 진동하는 애니메이션(`translateX` 핑퐁)과 반짝이는 글로우 효과가 짧게 재생됩니다.

## 5. Web Audio API 사운드 엔진 (하프 음색 합성)

mp3 파일 없이 `AudioContext`를 활용하여 맑고 영롱한 **하프(발현악기)** 소리를 합성합니다.
- **Plucked String 특성**: 어택(Attack)은 매우 짧고 날카로우며(0.01s), 디케이(Decay)는 리니어/지수 형태로 길게(2~4초) 감쇠해야 합니다.
- 음색 합성: `sine`파를 기본으로 하되, 풍성한 배음을 위해 `triangle` 파형을 섞고, 튕기는 순간의 마찰음을 위해 아주 짧은 노이즈(Lowpass Filter 통과) 버스트를 덧붙입니다.
- **잔향 효과**: `ConvolverNode`를 이용한 합성 임펄스 리버브를 추가하여, 마치 울림통이 있는 실제 하프처럼 소리가 공간에 은은하게 퍼지도록 만듭니다.
- 여러 현을 동시에 튕길 수 있도록 완전한 폴리포니(다중 발음)를 지원해야 합니다.
